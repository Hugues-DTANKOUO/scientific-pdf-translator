Mandatory pivot

What LangChain/LangGraph actually brings us

Separate the reason we're switching from the bonuses that come with it.

🎯 Core force — the reason for the shift

Universal multi-provider SDK

Native integration of OpenAI, Anthropic, Grok, and every emerging model. We're no longer locked into Azure AI. This is the essential point — the one thing we should focus on.

Bonus #1

Clear message types

Distinct roles in AI interactions (system, human, tool…).

Bonus #2

Native workflow & agents

Built-in tool orchestration in workflow or agent mode.

Bonus #3

Full control + flexibility

Write exactly the code you want where you need precision, delegate the rest to built-in components.

Bonus #4

Built-in components

Ready-made tools for repetitive tasks — already thought out and optimized for us.

These bonuses are real — but they're nice-to-haves, not the reason we're switching. We can forget them for now.
The architecture question

The sandwich problem

⚠ PromptFlow — redundant abstraction layer
LangChain / LangGraph — the new foundation
LLM providers · OpenAI · Anthropic · Grok · etc.
↑ Why maintain a layer that no longer serves a unique purpose?

If we only needed LangChain as an SDK, we could plug it into PromptFlow workflows and keep going. But once LangChain is our foundation, PromptFlow becomes a redundant wrapper — an extra layer of abstraction with no unique value left. It adds complexity, not capability.
